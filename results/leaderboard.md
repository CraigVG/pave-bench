# PaveBench Leaderboard

No public submissions yet. This file is generated by `pavebench leaderboard`
from a runs manifest (see `results/runs.example.jsonl`):

```bash
python3 -m pavebench.cli leaderboard --runs results/runs.jsonl --out results/leaderboard.md
```

Latency class: **INSTANT** = p95 wall-clock <= 10 s, **FAST** <= 60 s,
**BATCH** > 60 s (human-in-the-loop incumbents live here at 6-72 h).

The **INSTANT badge** is the load-bearing claim ("first verified instant AI
pavement takeoff"). A system earns it only when **all** of the following hold,
so the badge cannot be gamed by a fast-but-inaccurate or human-assisted system:

- Latency class INSTANT (p95 <= 10 s).
- Automation declared `full_auto` (no human in the loop).
- Pass rate >= 90% across >= 20 `reviewed_gold` cases spanning >= 2 GSD tiers.

Full methodology and the roadmap from one case to a publishable set:
[docs/proving-instant-takeoff.md](../docs/proving-instant-takeoff.md).

Tables are split by task (paved-area takeoff, stall counting) and disclose, per
system: pass rate, mean IoU, median/p90 area (sqft) delta, area-weighted cutout
recovery, p95 latency + latency class, mean cost per case, and the INSTANT badge.
